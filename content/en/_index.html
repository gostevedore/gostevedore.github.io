{{< /blocks/cover >}} {{% blocks/lead color="dark" %}} Stevedore is a tool for building Docker images at scale, and it is not intended to replace Dockerfile or Buildkit. Instead, Stevedore can be used in conjunction with these tools to help streamline the process of building and promoting multiple Docker images. {{% /blocks/lead %}} {{< blocks/section color="light" >}} {{% blocks/feature icon="fas fa-copy" title="Standardize" %}} Build your Docker images in a standardized way. Create a Dockerfile and reuse it for as many use cases as needed {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-project-diagram" title="Relatives build" %}} Automate the building of related Docker images by defining their relationships and dependencies {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-key" title="Credentials" %}} Store credentials to log in to your Docker registry, AWS Elastic Container Registry or a Git server {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-rocket" title="Promote" %}} Easily promote or copy Docker images from one registry to another using Stevedore's image promotion feature {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-file-alt" title="Semver awareness" %}} Automatically generate tags for semantic version 2.0.0 when building Docker image {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-box-open" title="Multiple build contexts" %}} Build your images using multiple build context sources. Use local files, a git repository or merge several sources {{% /blocks/feature %}} {{< /blocks/section >}}
